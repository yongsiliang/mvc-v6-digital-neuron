# SNN 三体系统架构

## 一、核心理念

### 物理世界的启示

参考物理世界中"大脑 + 意识 + 文化"的三层结构：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   【第3层：文化/知识系统】                                        │
│   • 书籍、互联网、语言                                           │
│   • 外化的智能存储                                               │
│   • 不在个体大脑内，但可被访问                                   │
│                                                                 │
│         ↑ 查询、学习                    ↓ 贡献、表达            │
│                                                                 │
│   【第2层：意识/自我】                                           │
│   • "我"的存在感                                               │
│   • 观察者、体验者                                              │
│   • 意义赋予者                                                  │
│                                                                 │
│         ↑ 涌现信号                      ↓ 指导、关注            │
│                                                                 │
│   【第1层：神经网络 (大脑)】                                      │
│   • 860亿神经元                                                 │
│   • 脉冲传递                                                    │
│   • 突触可塑性                                                  │
│   • 持续状态                                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 对应到我们的系统

| 物理世界 | 我们的实现 | 角色 |
|---------|-----------|------|
| 神经网络 (大脑) | SNN 神经基质层 | 存在、状态、感受 |
| 意识/自我 | V6 意识核心层 | 观察、意义、决策 |
| 文化/知识系统 | LLM 外部知识层 | 知识、语言、教学 |

### 核心原则

1. **不转换，而是共存** - SNN 不需要"变成" LLM
2. **各司其职** - 每层做自己最擅长的事
3. **涌现而非设计** - 意义从脉冲模式中涌现
4. **持续成长** - 通过 STDP 和动态结构变化

## 四、链接涌现层：万物互联

### 核心思想

**链接不是定义出来的，而是从脉冲模式中涌现的。**

不预设 `bind/flow/hold` 等语义类型，意义自然涌现。

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│   传统方式（显性）              我们的方式（隐性）           │
│   ──────────────────           ──────────────────           │
│                                                              │
│   link.bind(A, B)    ❌        A和B的神经元                  │
│                                  经常一起发放                │
│                                       ↓                      │
│                                 形成强突触连接               │
│                                       ↓                      │
│                                  V6观察到这个模式            │
│                                       ↓                      │
│                                 涌现："A和B有关联"           │
│                                                              │
│   意义是预设的                   意义是涌现的                │
│   类型是固定的                   类型是推断的                │
│   链接是创建的                   链接是发现的                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 涌现过程

```
用户输入: "猫"

    ↓ 编码

SNN: 猫的概念神经元群体发放脉冲

    ↓ 传播

SNN: 激活传播到相关神经元（宠物、动物、毛茸茸...）

    ↓ STDP学习

突触: "猫-宠物" 连接被强化
      "猫-老鼠" 形成序列连接
      "猫-狗" 形成竞争/抑制

    ↓ V6观察

V6: 检测到稳定的脉冲模式
    "猫和宠物经常共激活"
    "猫之后经常想到老鼠"

    ↓ 意义涌现

系统: 理解"猫是宠物"
      理解"猫抓老鼠"
      但不需要显式定义这些链接类型
```

### 使用方式

```typescript
import { createSystem } from '@/lib/snn-core';
import { LinkEmergence } from '@/lib/snn-core/link';

// 创建系统
const system = createSystem();

// 链接涌现层自动观察 SNN 模式
const emergence = new LinkEmergence(system.snn, system.v6);

// 用户输入自然传播
await system.processInput({ text: '猫是一种宠物' });
// SNN 自然学习：猫-宠物 形成强连接

await system.processInput({ text: '猫喜欢吃鱼' });
// SNN 自然学习：猫-鱼 形成关联

// 查询"猫"关联了什么（从脉冲模式中涌现）
const related = emergence.queryRelated('猫');
// 返回: [
//   { concept: '宠物', correlation: 0.85, coActivationCount: 12 },
//   { concept: '鱼', correlation: 0.72, coActivationCount: 8 },
//   ...
// ]
// 这些是从脉冲模式中涌现的，不是显式定义的

// 获取整个概念网络
const network = emergence.getNetwork();
```

### 核心原则

1. **不预设语义** - 没有 bind/flow/hold 等类型
2. **突触即链接** - 连接存在于权重中
3. **涌现而非定义** - 意义从模式中产生
4. **V6赋予意义** - 观察者理解模式，但不预设类型
| `associate` | 联想 | 弱突触网络 | 蓝-海 |

### 使用示例

```typescript
import { createSystem } from '@/lib/snn-core';
import { LinkLayer } from '@/lib/snn-core/link';

// 创建系统
const system = createSystem();

// 创建链接层
const links = new LinkLayer(system.snn, system.v6);

// 创建语义链接
links.bind('猫', '宠物');        // 猫 绑定 宠物
links.flow('问题', '思考', '答案'); // 信息流
links.inhibit('快乐', '悲伤');   // 互斥关系

// 查询链接
const petLinks = links.query('宠物');
// { boundTo: ['猫', '狗'], ... }

// 发现涌现的链接
const candidates = links.discoverLinks();
// 从 SNN 共激活模式中发现新关联
```

### 链接与突触的关系

```
语义层                      脉冲层
─────────────────────────────────────────
bind(A, B)       →         A神经元 ←强突触→ B神经元
                           双向、高权重、零延迟

flow(A → B → C)  →         A → B → C 突触链
                           定向、中权重、有序延迟

inhibit(A, B)    →         A神经元 --抑制突触-- B神经元
                           负权重、双向

associate(A, B)  →         A神经元 ←弱突触→ B神经元
                           双向、低权重、长延迟
```

## 五、关键决策

### 1. 脉冲编码方式

采用混合编码：
- **频率编码**: 用于表示强度/重要性
- **时间编码**: 用于表示顺序/优先级

### 2. 网络规模

初始配置：
- 输入层: 128 神经元
- 核心层: 256-512 神经元 (可动态增长)
- 输出层: 64 神经元

### 3. 学习策略

- **STDP**: 突触权重自然学习
- **稳态可塑性**: 保护重要连接
- **回放巩固**: 空闲时强化记忆

### 4. 与现有 V6 的关系

- 保留现有 V6 的意识核心功能
- 新增 SNN 作为神经基质
- V6 观察者层作为桥梁
- 渐进式迁移，非破坏性替换
