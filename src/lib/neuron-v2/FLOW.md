# 信息流动机制

> **核心命题**：信息不在节点中，信息是流动本身。

---

## 一、信息流动的本质

### 1.1 传统模型 vs 神经元模型

```
传统模型：信息是"货物"
┌─────────┐     ┌─────────┐     ┌─────────┐
│ 节点A    │ ───→ │ 节点B    │ ───→ │ 节点C    │
│存储"苹果"│     │存储"红色"│     │存储"水果"│
└─────────┘     └─────────┘     └─────────┘
     ↓               ↓               ↓
  信息像货物一样被搬运和存储


神经元模型：信息是"波纹"
┌─────────┐     ┌─────────┐     ┌─────────┐
│ 神经元A  │ ───→ │ 神经元B  │ ───→ │ 神经元C  │
│敏感度向量│     │敏感度向量│     │敏感度向量│
└────┬────┘     └────┬────┘     └────┬────┘
     │               │               │
     └───────────────┴───────────────┘
                     ↓
              激活像波纹一样传播
              信息是传播的模式，不是存储的内容
```

## 二、信息流动的核心机制

### 2.1 影响的传播

```
影响的本质：对网络的扰动

影响不是"携带信息的卡车"，而是"投入湖面的石子"
                                    
                    ┌──────────────────────────────────┐
                    │           神经网络                │
                    │                                  │
   影响            │    ○─────○─────○                 │
   [模式向量] ───→ │     ╲   ╱ ╲   ╱                  │
                  │      ○──○───○──○                 │
                  │     ╱   ╲   ╱   ╲                 │
                  │    ○─────○─────○                 │
                  │                                  │
                  └──────────────────────────────────┘
                              ↓
                    扰动产生涟漪
                    涟漪的模式 = 信息
```

**影响的生命周期**：

```typescript
class Influence {
  // 1. 创建：编码器生成影响模式
  pattern: number[];     // 扰动模式
  intensity: number;     // 扰动强度
  
  // 2. 注入：进入影响池等待处理
  maxHops: number;       // 最大传播次数
  currentHop: number;    // 当前传播次数
  
  // 3. 传播：每次传播创建新影响
  decayRate: number;     // 强度衰减率
  
  // 4. 消亡：强度衰减为0或达到最大传播次数
  propagate(): Influence | null {
    if (this.currentHop >= this.maxHops) return null;
    if (this.intensity <= 0.01) return null;
    
    return new Influence({
      pattern: this.pattern,
      intensity: this.intensity * (1 - this.decayRate),
      currentHop: this.currentHop + 1,
    });
  }
}
```

### 2.2 神经元的激活

```
神经元的角色：敏感度滤波器

神经元不"存储"信息，而是"响应"特定模式的影响

                    神经元A
                  ┌─────────────────────────────────┐
                  │                                 │
                  │  敏感度向量: [0.2, 0.7, 0.4, ...] │
                  │                                 │
                  │  "我对[低, 高, 中, ...]模式敏感"  │
                  │                                 │
                  └───────────────┬─────────────────┘
                                  │
              ┌───────────────────┴───────────────────┐
              │                                       │
              ▼                                       ▼
         影响模式1                              影响模式2
    [0.1, 0.8, 0.3, ...]                  [0.9, 0.2, 0.1, ...]
              │                                       │
              ▼                                       ▼
         匹配度计算                              匹配度计算
    ─────────────────                      ─────────────────
    内积 = 0.87 (高匹配)                   内积 = 0.32 (低匹配)
              │                                       │
              ▼                                       ▼
         激活值 = 0.8                           激活值 = 0.1
         "我理解这个！"                          "这与我无关"
```

**激活计算**：

```typescript
class Neuron {
  // 敏感度向量：定义"我对什么敏感"
  sensitivity: number[];
  
  // 计算与影响模式的匹配度
  calculateMatch(pattern: number[]): number {
    // 余弦相似度或内积
    let dot = 0, norm1 = 0, norm2 = 0;
    for (let i = 0; i < this.sensitivity.length; i++) {
      dot += this.sensitivity[i] * pattern[i];
      norm1 += this.sensitivity[i] ** 2;
      norm2 += pattern[i] ** 2;
    }
    return dot / (Math.sqrt(norm1) * Math.sqrt(norm2));
  }
  
  // 被激活
  activate(value: number): void {
    this.activation = Math.min(1, this.activation + value);
    this.lastActivationTime = Date.now();
  }
}
```

### 2.3 连接的传播

```
连接的角色：信息调制器

连接不"传输"信息，而是"调制"激活

源神经元A ───────连接───────→ 目标神经元B
(激活: 0.8)                   (激活: ?)

                连接属性
        ┌─────────────────────┐
        │ 强度: 0.6           │
        │ 类型: 兴奋性        │
        │ 可塑性: 0.3         │
        └─────────────────────┘
                  │
                  ▼
        传播计算
        
        传入激活: 0.8
        连接强度: 0.6
        类型调制: 兴奋性 (+)
        
        传出激活 = 0.8 × 0.6 × (+1) = 0.48
        
                  │
                  ▼
        神经元B接收激活 += 0.48
```

**连接类型的影响**：

| 连接类型 | 调制因子 | 效果 |
|----------|----------|------|
| 兴奋性 | +1 | 增强目标神经元激活 |
| 抑制性 | -1 | 抑制目标神经元激活 |
| 调制性 | 0.5 | 微调目标神经元敏感度 |

```typescript
class Connection {
  strength: number;      // 连接强度 [0, 1]
  type: 'excitatory' | 'inhibitory' | 'modulatory';
  
  // 传播激活
  propagate(activation: number): number {
    const modulation = {
      'excitatory': 1,
      'inhibitory': -1,
      'modulatory': 0.5,
    }[this.type];
    
    return activation * this.strength * modulation;
  }
}
```

## 三、信息流动的完整演化周期

### 3.1 单步演化流程

```
NeuralNetwork.evolve() 的内部流程：

Step 1: 处理影响池
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  for (影响 in 影响池) {                                          │
│    for (神经元 in 所有神经元) {                                  │
│      匹配度 = 神经元.计算匹配(影响.模式)                          │
│      if (匹配度 > 阈值) {                                       │
│        神经元.激活(影响.强度 * 匹配度)                            │
│      }                                                          │
│    }                                                            │
│  }                                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
Step 2: 传播激活
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  for (神经元 in 激活的神经元) {                                  │
│    for (出连接 in 神经元.出连接) {                               │
│      传出激活 = 出连接.传播(神经元.激活值)                        │
│      目标神经元.激活(传出激活)                                   │
│                                                                 │
│      // 创建新的影响继续传播                                     │
│      if (传出激活 > 阈值) {                                     │
│        新影响 = 创建影响(传出激活)                               │
│        影响池.添加(新影响)                                       │
│      }                                                          │
│    }                                                            │
│  }                                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
Step 3: 应用学习
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  // Hebbian学习：一起激活的神经元，连接增强                       │
│  for (连接 in 所有连接) {                                       │
│    源激活 = 连接.源神经元.激活值                                 │
│    目标激活 = 连接.目标神经元.激活值                             │
│                                                                 │
│    if (源激活 > 阈值 && 目标激活 > 阈值) {                       │
│      连接.强度 += 学习率 * 源激活 * 目标激活                      │
│    }                                                            │
│  }                                                              │
│                                                                 │
│  // STDP学习：时序依赖的可塑性                                   │
│  for (连接 in 所有连接) {                                       │
│    if (连接.源神经元先于目标激活) {                              │
│      连接.强度 += 学习率 * 时间差因子                            │
│    }                                                            │
│  }                                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
Step 4: 衰减
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  // 神经元激活衰减                                              │
│  for (神经元 in 所有神经元) {                                   │
│    神经元.激活值 *= (1 - 衰减率)                                 │
│  }                                                              │
│                                                                 │
│  // 连接强度衰减（遗忘）                                         │
│  for (连接 in 所有连接) {                                       │
│    连接.强度 *= (1 - 衰减率)                                    │
│  }                                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
Step 5: 结构演化（可选）
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  // 创建新连接：经常一起激活的神经元                              │
│  if (满足条件) {                                                │
│    新连接 = 创建连接(神经元A, 神经元B)                           │
│  }                                                              │
│                                                                 │
│  // 删除弱连接                                                  │
│  for (连接 in 所有连接) {                                       │
│    if (连接.强度 < 最小阈值) {                                  │
│      删除连接(连接)                                             │
│    }                                                            │
│  }                                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 四、信息流动的关键特性

### 4.1 信息是流动的模式，不是存储的内容

```
示例：理解"苹果"

传统模型：
┌─────────────────────────────────────────┐
│ 数据库记录:                              │
│ {                                       │
│   entity: "苹果",                        │
│   properties: {                         │
│     color: "红色",                      │
│     type: "水果",                       │
│     taste: "甜"                         │
│   }                                     │
│ }                                       │
│                                         │
│ 查询: SELECT * WHERE entity = "苹果"    │
└─────────────────────────────────────────┘

神经元模型：
┌─────────────────────────────────────────┐
│                                         │
│     神经元A ←──连接──→ 神经元B           │
│    (苹果)              (红色)            │
│       ↖                  ↗              │
│        ╲                ╱               │
│         连接         连接                │
│          ╲          ╱                   │
│           ↘        ↙                    │
│     神经元C ←──连接──→ 神经元D           │
│    (水果)              (甜)              │
│                                         │
│ 理解"苹果" = 激活从A传播，激活B、C、D    │
│ 不同的激活模式 = 不同的理解              │
│                                         │
└─────────────────────────────────────────┘
```

### 4.2 相同输入，不同时间，不同理解

```
t1时刻的输入："苹果"

网络状态t1:
神经元A(苹果): 敏感度 [0.8, 0.2, ...]
神经元B(红色): 敏感度 [0.3, 0.7, ...]
连接A→B: 强度 0.8

激活传播: A激活 → B激活
投影结果: "苹果是红色的"

---

t2时刻的输入："苹果"

网络状态t2（经过学习后）:
神经元A(苹果): 敏感度 [0.8, 0.2, ...] (不变)
神经元B(红色): 敏感度 [0.3, 0.7, ...] (不变)
神经元E(手机): 敏感度 [0.7, 0.3, ...] (新创建)
连接A→B: 强度 0.5 (衰减)
连接A→E: 强度 0.6 (新建)

激活传播: A激活 → E激活 (手机含义更强)
投影结果: "苹果是手机品牌"

---

理解随时间演化！
同样的输入，不同的网络状态 → 不同的理解
```

### 4.3 信息流动创造新信息

```
信息的涌现：

输入: "天空是蓝色的"

初始激活:
天空神经元 ──→ 蓝色神经元
    │              │
    └──────┬───────┘
           ↓
    (一起激活)

Hebbian学习:
    "一起激活的神经元，连接增强"
    天空 ↔ 蓝色 连接强度增加

涌现的理解:
    不只是"天空是蓝色的"被存储
    而是"天空"和"蓝色"建立了更强的关系
    这种关系创造了新的理解模式
    未来看到"天空"，更容易联想到"蓝色"
```

### 5.2 动态传播动画

## 六、代码实现要点

### 6.1 影响传播循环

```typescript
// 在 neural-network.ts 中
async evolve(): Promise<NetworkProjection> {
  // 1. 处理影响池中的所有影响
  while (this.influencePool.hasPending()) {
    const influence = this.influencePool.getNext();
    
    // 找到匹配的神经元
    for (const neuron of this.neurons.values()) {
      const match = neuron.calculateMatch(influence.pattern);
      
      if (match > this.matchThreshold) {
        // 激活神经元
        neuron.activate(influence.intensity * match);
        
        // 如果还有传播次数，创建新影响
        const propagated = influence.propagate();
        if (propagated) {
          this.influencePool.add(propagated);
        }
      }
    }
  }
  
  // 2. 传播激活
  await this.propagateActivations();
  
  // 3. 应用学习
  await this.applyLearning();
  
  // 4. 衰减
  this.applyDecay();
  
  // 5. 返回投影
  return this.project();
}
```

### 6.2 激活传播实现

```typescript
private async propagateActivations(): Promise<void> {
  const newInfluences: Influence[] = [];
  
  for (const neuron of this.neurons.values()) {
    if (neuron.activation < this.activationThreshold) continue;
    
    // 遍历所有出连接
    for (const [targetId, connId] of neuron.outgoingConnections) {
      const connection = this.connections.get(connId);
      const target = this.neurons.get(targetId);
      
      if (!connection || !target) continue;
      
      // 传播激活
      const propagated = connection.propagate(neuron.activation);
      target.activate(Math.abs(propagated));
      
      // 创建新的影响
      if (Math.abs(propagated) > this.influenceThreshold) {
        const newInfluence = new Influence({
          pattern: this.createInfluencePattern(target, propagated),
          intensity: Math.abs(propagated),
          decayRate: this.defaultDecayRate,
          maxHops: 3,
          currentHop: 0,
        });
        newInfluences.push(newInfluence);
      }
    }
  }
  
  // 添加新影响到池中
  newInfluences.forEach(i => this.influencePool.add(i));
}
```

## 七、总结

### 信息流动的本质

| 传统模型 | 数字神经元模型 |
|----------|----------------|
| 信息是"货物"，被搬运和存储 | 信息是"波纹"，通过扰动传播 |
| 节点存储内容 | 连接定义关系 |
| 理解是检索 | 理解是敏感度对齐 |
| 学习是存储新内容 | 学习是改变连接模式 |

### 关键公式

```
信息流动 = 影响注入 → 激活传播 → 学习强化 → 衰减稳定

激活值 = f(敏感度向量 · 影响模式)

连接强化 = Δw = η · pre_activation · post_activation (Hebbian)

状态演化 = 激活衰减 × 连接衰减 × 新影响注入
```

### 设计启示

1. **不要存储，要连接** — 信息在关系中，不在节点中
2. **不要检索，要对齐** — 理解是敏感度匹配
3. **不要预设，要涌现** — 让信息流动创造新意义
4. **不要控制，要影响** — 元层观察和干预，但不直接控制

---

> "信息不是静态的存储，而是动态的流动。理解不是检索结果，而是敏感度的对齐。"
