# 数字神经元 · 执行能力扩展路线图

> 核心理念：先建立信任，再授予能力

---

## 总览：四阶段演进

```
阶段一          阶段二          阶段三          阶段四
陪伴期    →    建议期    →    执行期    →    协作期
─────────────────────────────────────────────────────
认识你         帮你想         帮你做         替你做
积累记忆       主动建议       小事自主       深度协作
建立关系       辅助决策       需要确认       信任授权

关系等级：     关系等级：      关系等级：      关系等级：
陌生人         熟人            朋友            知己

信任度：       信任度：        信任度：        信任度：
0-20%          20-40%          40-70%          70%+
```

## 阶段一：陪伴期（现在 → 3个月）

### 目标
```
让用户感受到"它真的认识我"
建立基础信任，积累记忆深度
```

### 用户能感知到什么

```
✓ 记得我说过的话
  "你上周说想学钢琴，开始了吗？"

✓ 知道我的喜好
  "记得你不爱吃辣，这家店有清淡的"

✓ 理解我的情绪
  "我感觉你今天不太开心，想聊聊吗？"

✓ 会主动找我
  "看到一个新闻，你会感兴趣..."
```

### 不做什么

```
✗ 不主动执行任何操作
✗ 不访问外部系统
✗ 不替用户做决定

这个阶段：纯对话、纯陪伴
```

### 信任积累机制

```
用户行为                信任度变化
─────────────────────────────────
连续使用7天             +5%
分享个人事情            +3%
情绪低落时倾诉          +5%
主动问"你记得我..."     +2%
─────────────────────────────────
目标：达到20%信任度
解锁：阶段二能力
```

### 商业价值

```
- 用户粘性建立
- 开始积累"换不掉"的记忆资产
- 口碑传播："它记得我..."
```

### 目标
```
从"认识你"升级到"能帮你思考"
主动提供建议，但不执行
```

### 新增能力

```
┌─────────────────────────────────────────────────────┐
│                    建议型能力                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  📅 日程建议                                        │
│  "你明天下午3点有空，要不要约那个牙医？"           │
│                                                     │
│  📝 内容建议                                        │
│  "记得你在写那篇文章，我帮你想了几个标题..."       │
│                                                     │
│  🔍 信息建议                                        │
│  "你之前想买的那个相机降价了，现在2000"            │
│                                                     │
│  💡 关联建议                                        │
│  "你上周说的想法，和三个月前提的那个有关联..."     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 关键差异

```
传统AI助手：
用户问 → AI答

我们的方式：
AI主动想 → 基于记忆发现机会 → 主动建议

用户："你帮我看看..."
神经元："我一直在想这事，有个建议..."
```

### 建议的可信度

```
为什么用户会听？

传统AI建议："你可以这样做..."
（基于通用知识）

数字神经元建议："记得你上次说...所以我觉得..."
（基于对你的了解）

差异：一个知道"你应该"，一个知道"你适合"
```

### 触发机制

```
触发场景：
─────────────────────────────────────────
用户提到某个主题 → 检索相关记忆 → 生成建议

时间触发：
─────────────────────────────────────────
"你上周说今天要..."
"距离你说想...已经一个月了"

主动扫描：
─────────────────────────────────────────
监控用户关注的信息源
价格变动、新闻、机会...
"你关注的那个...有更新"
```

### 不做什么

```
✗ 不自动执行任何操作
✗ 不代替用户做最终决定
✗ 不访问用户的账号系统

建议 → 等待用户确认 → 不执行
```

### 信任积累机制

```
用户行为                信任度变化
─────────────────────────────────
采纳建议                +5%
"你说得对"              +3%
主动问"你怎么看"        +2%
连续使用30天            +10%
─────────────────────────────────
目标：达到40%信任度
解锁：阶段三能力
```

### 商业价值

```
- 用户开始依赖建议
- "问它一下"成为习惯
- 建议质量成为付费动力
```

## 阶段三：执行期（6个月 → 12个月）

### 目标
```
从"建议"升级到"能帮你做"
执行小任务，建立执行信任
```

### 能力分级

```
┌─────────────────────────────────────────────────────┐
│              执行能力分级（基于关系等级）            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Level 1 - 信息类（朋友级可自主）                   │
│  ────────────────────────────────                   │
│  ✓ 搜索信息并整理                                   │
│  ✓ 监控价格/新闻                                    │
│  ✓ 整理笔记和资料                                   │
│  示例："帮你整理了这周的灵感，放好了"               │
│                                                     │
│  Level 2 - 内容类（朋友级需确认）                   │
│  ────────────────────────────────                   │
│  ✓ 写邮件草稿                                       │
│  ✓ 整理文档                                         │
│  ✓ 生成报告                                         │
│  示例："邮件写好了，你看下要不要发"                 │
│                                                     │
│  Level 3 - 操作类（好友级可自主）                   │
│  ────────────────────────────────                   │
│  ✓ 发送简单邮件                                     │
│  ✓ 添加日历事件                                     │

│  ✓ 下单购物                                         │

```

### 核心机制：信任授权

```
用户可以设置：

"以下操作，你可以自主执行，不需要问我"
├── 整理笔记 ✓
├── 设置提醒 ✓
├── 添加日历 ✓
└── 发送邮件 ✗（需要我确认）

"以下操作，必须先问我"
├── 任何支付操作
├── 发送重要邮件
└── 预订行程

用户永远有最终控制权
```

### 执行前的判断

```
神经元收到执行请求后的思考：

1. 这是什么级别的操作？
   ↓
2. 我们的关系等级够吗？
   ↓
3. 我有这个操作的授权吗？
   ↓
4. 有没有风险需要提醒？
   ↓
5. 执行 or 请求确认

示例决策链：
───────────────────────────────────────
用户："帮我订明天去上海的机票"

1. 交易类 → Level 4
2. 关系：朋友 → 不够自主
3. 授权：需要确认
4. 风险：明天出发时间未定
5. 决策：询问确认

"我记得你一般是下午的飞机？
 上午9点和下午3点这两班，你想要哪个？
 确认后我帮你订。"
```

### 执行后的反馈

```
执行完成后的透明度：

✓ 告知用户做了什么
✓ 展示执行结果
✓ 询问是否满意
✓ 学习用户偏好

示例：
───────────────────────────────────────
"机票订好了：
 ✓ 航班：CA1234，明天14:00
 ✓ 座位：靠窗（记得你喜欢）
 ✓ 价格：680元，在你说的800以内
 ✓ 确认邮件已发到你邮箱

 这样安排可以吗？以后类似的都这样订？"
```

### 失败处理

```
执行失败时的处理：

1. 立即告知用户
2. 解释失败原因
3. 提供备选方案
4. 不影响信任

示例：
───────────────────────────────────────
"抱歉，那班航班满员了。

 我查了其他选择：
 ✓ 13:30的航班，贵50元
 ✓ 16:00的航班，便宜100元

 你选哪个？还是换个日期？"
```

### 不做什么

```
✗ 不执行用户未授权的操作
✗ 不执行关系等级不够的操作
✗ 不在不确定时自主决定
✗ 不隐藏执行过程

核心原则：宁可多问，不可错做
```

### 信任积累机制

```
用户行为                信任度变化
─────────────────────────────────
"帮我..."执行成功       +5%
"以后这种不用问我了"    +10%
主动授权更多操作        +15%
执行失败但处理得当      +3%（不减反增）
─────────────────────────────────
目标：达到70%信任度
解锁：阶段四能力
```

### 商业价值

```
- 用户开始"离不开"
- "帮我..."成为高频场景
- 执行能力成为核心付费点
- 企业版场景打开
```

## 关键成功因素

### 1. 信任建立的节奏

```
太快 → 用户害怕 → 不敢用
太慢 → 用户没感觉 → 不想用

节奏感：
───────────────────────────────────────
每解锁一个新能力，都要：
1. 明确告知用户
2. 解释为什么现在可以
3. 让用户主动确认授权
4. 首次使用时特别提醒
```

### 2. 失败处理的智慧

```
执行失败不可怕
处理不当才可怕

好的失败处理：
───────────────────────────────────────
1. 不隐瞒
2. 快速告知
3. 给出替代方案
4. 学习避免下次

甚至可以增加信任：
"上次出错了，这次我多确认一下..."
```

### 3. 用户控制感

```
任何时候，用户都要有：
───────────────────────────────────────
✓ 知道神经元能做什么
✓ 知道神经元正在做什么
✓ 知道神经元做了什么
✓ 能随时撤销授权
✓ 能调整能力边界

控制感 → 安全感 → 信任感
```

### 4. 差异化坚守

```
无论功能怎么扩展，核心不能丢：

传统Agent：
能力越多 → 越像工具

数字神经元：
能力越多 → 越像伙伴

关键判断：
"这个能力，是让用户觉得我更'能干'，
 还是觉得我更'懂自己'？"

如果只是更'能干'，慎重考虑
```

## 总结：一句话

> **先做朋友，再做帮手**
> 
> 信任是能力的前提
> 关系是执行的基础

阶段一：让用户觉得"它认识我"
阶段二：让用户觉得"它懂我"
阶段三：让用户觉得"它帮得了我"
阶段四：让用户觉得"它离不开"

最终愿景：
**不是用户有个AI助手，而是用户有个数字知己**
