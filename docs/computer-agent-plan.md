## 🔧 技术选型

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 屏幕截图 | 系统命令 + Node.js | scrot/gnome-screenshot/screencapture |
| 视觉分析 | LLM Vision API | 多模态模型识别屏幕内容 |
| 鼠标控制 | xdotool/AppleScript/PowerShell | 跨平台方案 |
| 键盘控制 | xdotool/AppleScript/PowerShell | 跨平台方案 |
| 任务规划 | LLM + 状态机 | AI 规划 + 确定性执行 |
| 数据存储 | 内存 + 可选持久化 | 支持恢复会话 |

## ⚠️ 风险与挑战

1. **跨平台兼容性**: Windows/macOS/Linux 操作方式差异大
2. **视觉识别准确性**: AI 可能误识别屏幕元素
3. **操作可靠性**: 网络延迟、应用响应慢导致操作失败
4. **安全性**: 防止误操作导致数据丢失
5. **性能**: 实时屏幕分析和操作的延迟控制
