# 阴阳双系统架构修正：LLM作为阳系统核心

## 一、问题诊断

### 1.1 当前架构（错误）

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────┐
│         NeuronSystemV3.processInput()        │
│                                             │
│   ┌─────────────┐    ┌─────────────┐        │
│   │ 阴系统       │    │ 阳系统       │        │
│   │ Hebbian网络  │    │ VSA空间      │ ← 问题！│
│   │ (直觉联想)   │    │ (语义向量)   │        │
│   └─────────────┘    └─────────────┘        │
│                                             │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────┐
│      LLM        │  ← 只是"表达器"，不参与阴阳互塑
│   (生成回复)     │
└─────────────────┘
```

**问题**：
1. VSA空间不是"阳系统"，它只是语义表示工具
2. LLM作为输出工具，没有参与核心认知处理
3. 阴阳互塑不涉及真正的理性推理

### 1.2 正确架构

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                      YinYangBridge                          │
│                                                             │
│   ┌─────────────────────┐       ┌─────────────────────┐    │
│   │      阴系统          │       │      阳系统          │    │
│   │   Hebbian Network    │◄─────►│        LLM          │    │
│   │                      │ 互塑  │                     │    │
│   │  • 直觉联想 (快)      │       │  • 理性推理 (慢)     │    │
│   │  • 无意识            │       │  • 有意识           │    │
│   │  • 分布式表示        │       │  • 符号推理         │    │
│   │  • 动态可塑          │       │  • 稳定知识         │    │
│   │                      │       │                     │    │
│   │  系统1 (Kahneman)    │       │  系统2 (Kahneman)   │    │
│   └─────────────────────┘       └─────────────────────┘    │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                    Self Core                         │  │
│   │                    同一性载体                         │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
融合输出（直觉 + 理性）
```

## 三、代码修改计划

### 3.1 YinYangBridge 修改

```typescript
// 修改前：阳系统 = VSA空间
yangContribution = this.vsaSpace.findSimilarConcepts(inputVector);

// 修改后：阳系统 = LLM调用
yangContribution = await this.callLLMForYangSystem(input, yinContribution);
```

### 3.2 route.ts 数据流修改

```typescript
// 修改前：神经元处理 → LLM生成回复
const neuronResult = await neuronSystem.processInput(message);
// LLM只是表达器...

// 修改后：阴阳互塑在LLM调用中完成
const yinYangResult = await yinYangBridge.processWithLLM(message, {
  // 阴系统先激活
  yinActivation: hebbianNetwork.spreadActivation(inputConcepts),
  
  // 阳系统(LLM)结合阴系统直觉进行推理
  yangReasoning: await llmClient.stream(messages, {
    systemPrompt: buildPromptWithYinContext(yinActivation)
  }),
  
  // 阳系统结果塑造阴系统
  shapingCallback: (llmOutput) => {
    hebbianNetwork.strengthenFromReasoning(extractConcepts(llmOutput));
  }
});
```
