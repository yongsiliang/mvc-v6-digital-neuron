## 一、核心发现

### 1.1 贾维斯的数学本质

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   J = (U, W, π)                                                        │
│                                                                         │
│   其中:                                                                 │
│   • U = User State (用户状态)                                          │
│   • W = World State (世界状态)                                         │
│   • π = Policy (策略函数)                                              │
│                                                                         │
│   核心循环:                                                             │
│   process(m) = respond(execute(plan(recall(understand(m)))))           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 二、三个核心数据结构

### 2.1 User State (用户状态)

```typescript
interface UserState {
  // P: 偏好向量
  preferences: {
    vector: number[];        // 可学习的偏好向量
    explicit: Map<string, PreferenceEntry>;
    implicit: Map<string, PreferenceEntry>;
  };
  
  // G: 目标树
  goals: {
    active: Goal[];
    completed: Goal[];
    abandoned: Goal[];
  };
  
  // H: 经验轨迹 (强化学习)
  experience: {
    recent: Experience[];
    significant: Experience[];
    stats: { totalInteractions, averageSatisfaction, topicsDiscussed };
  };
  
  // R: 关系网络
  relationships: {
    people: Relationship[];
    entities: EntityRelation[];
    withAI: { trustLevel, familiarity, interactionStyle };
  };
}
```

**本质洞察**: 用户状态不是对话历史的堆砌，而是对用户的**完整建模**。

### 2.2 World State (世界状态)

```typescript
interface WorldState {
  // E: 实体
  entities: Map<string, WorldEntity>;
  
  // R: 关系
  relations: WorldRelation[];
  
  // O: 可观察状态
  observable: { time, timezone, date, device?, location? };
  
  // A: 可用动作
  availableActions: {
    information: ActionCapability[];
    execution: ActionCapability[];
    communication: ActionCapability[];
  };
}
```

**本质洞察**: 世界状态不是工具列表，而是对世界的**完整建模**。

### 2.3 System Policy (系统策略)

```typescript
interface SystemPolicy {
  // 核心策略函数
  understandIntent: (message, userState) => Intent;
  recallMemory: (intent, userState) => MemoryContext;
  planActions: (intent, context, worldState) => Action[];
  executeAction: (action) => ActionResult;
  generateResponse: (intent, results, context, userState) => string;
  updateState: (interaction) => StateUpdate;
  
  // 学习机制
  learn: (experience) => void;
  getParameters: () => PolicyParameters;
  setParameters: (params) => void;
}
```

**本质洞察**: 系统策略不是规则引擎，而是**可学习的决策函数**。

## 三、核心循环

### 3.1 处理流程

```
用户消息 m
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Step 1: understandIntent(m, U)                                     │
│  ───────────────────────────────                                    │
│  输入: 消息 + 用户状态                                               │
│  输出: 结构化意图                                                    │
│  问题: 用户想要什么？                                                │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Step 2: recallMemory(intent, U)                                    │
│  ───────────────────────────────                                    │
│  输入: 意图 + 用户状态                                               │
│  输出: 相关上下文                                                    │
│  问题: 相关的历史和偏好是什么？                                      │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Step 3: planActions(intent, context, W)                            │
│  ───────────────────────────────                                    │
│  输入: 意图 + 上下文 + 世界状态                                      │
│  输出: 动作序列                                                      │
│  问题: 应该做什么？                                                  │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Step 4: executeAction(action)                                      │
│  ───────────────────────────────                                    │
│  输入: 动作                                                          │
│  输出: 执行结果                                                      │
│  问题: 结果是什么？                                                  │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Step 5: generateResponse(intent, results, context, U)              │
│  ───────────────────────────────                                    │
│  输入: 意图 + 结果 + 上下文 + 用户状态                               │
│  输出: 响应文本                                                      │
│  问题: 如何回复用户？                                                │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  Step 6: updateState(interaction)                                   │
│  ───────────────────────────────                                    │
│  输入: 完整交互                                                      │
│  输出: 状态更新                                                      │
│  问题: 学到了什么？                                                  │
└─────────────────────────────────────────────────────────────────────┘
    │
    ▼
响应 + 状态更新
```

### 3.2 关键设计决策

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       关键设计决策                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 单一入口                                                            │
│     ─────────────                                                      │
│     只有一个 process(message) 方法                                      │
│     所有复杂性都被封装在内部                                            │
│                                                                         │
│  2. 状态驱动                                                            │
│     ─────────────                                                      │
│     系统行为完全由状态决定                                              │
│     相同输入 + 相同状态 = 相同输出                                      │
│                                                                         │
│  3. 可学习                                                              │
│     ─────────────                                                      │
│     策略参数可以通过反馈调整                                            │
│     经验轨迹用于持续改进                                                │
│                                                                         │
│  4. 可持久化                                                            │
│     ─────────────                                                      │
│     状态可以导出/导入                                                   │
│     支持跨会话记忆                                                      │
│                                                                         │
│  5. 可验证                                                              │
│     ─────────────                                                      │
│     每个步骤都可以独立测试                                              │
│     明确的输入输出类型                                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
