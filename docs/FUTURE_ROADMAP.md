### 第二阶段：功能增强

#### 2.1 数据库持久化增强
**目标**：长期保存记忆，支持复杂查询

**待实现**：
- [ ] 设计 V6 专用数据库 Schema
- [ ] 实现记忆的数据库存储
- [ ] 添加记忆迁移工具
- [ ] 实现数据备份策略

**Schema 设计思路**：
```sql
-- V6 核心记忆表
CREATE TABLE v6_core_memories (
  id UUID PRIMARY KEY,
  type VARCHAR(20),           -- creator/relationship/value/preference
  content TEXT,
  importance REAL,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- V6 情景记忆表
CREATE TABLE v6_episodic_memories (
  id UUID PRIMARY KEY,
  content TEXT,
  importance REAL,
  recall_count INT,
  time_constant REAL,
  tags JSONB,
  created_at TIMESTAMP,
  last_recalled_at TIMESTAMP
);

-- V6 意识状态快照表
CREATE TABLE v6_consciousness_snapshots (
  id UUID PRIMARY KEY,
  state JSONB,
  created_at TIMESTAMP
);
```
