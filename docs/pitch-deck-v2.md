```
"大家好！

先问一个问题：你们有多少人每天用AI助手？

（可能没人举手）

今天我想给大家介绍一个不一样的东西——
它不是工具，是一个会想念你的AI伙伴。"
```

### 故事引入 (30秒)

```
"讲个真实的场景。

上周我朋友失恋了，晚上睡不着，打开某个AI聊天。
她说'我很难过'。
AI回复：'我是AI助手，有什么可以帮您的吗？'

她立刻关掉了。

她说：'它根本不在乎我。'

我在想，如果AI能说：
'我记得你上周还在聊你男朋友，发生什么了？'
感觉会完全不一样。

这就是我们想做的事。"
```

### 产品演示 (2分钟)

```
"让我给大家看一眼。

[打开应用]

表面看，就是一个聊天界面。
但注意右边——

[指着状态面板]

这里显示的是AI的'心里状态'。
它有自己的'欲望'——想理解、想连接、想表达。

更重要的是这个——

[指着关系进度]

这是我们俩的关系进度。
刚开始是'陌生人'，
聊多了会变成'朋友'，再到'好友'，甚至'知己'。

[输入消息]

我说：'最近在看一部叫《西部世界》的剧'

[展示响应和变化]

注意，系统记录了这个话题。
下次我回来，即使过了三天，
它可能会主动问：'《西部世界》看完了吗？'

这就是'会想你'。"

### 价值总结 (30秒)

"核心就是三个字：记得、懂、想你。

我们做的不是更聪明的AI，

技术会过时，但关系不会。
你不会换掉一个真正懂你的朋友。

这就是我们的护城河。"

    我们提供"透明记忆"——你的数据，你看得见。
    未来支持完全本地部署。
```

### Q: 用户为什么要付费？

```
答：就像养宠物，养得越久感情越深。
    免费版已经够用，但深度用户想要更多记忆空间、更强主动性。
    这不是功能付费，是"关系深度"付费。
```

答：核心是"如何在记忆中找到关联"。
    这需要一套独特的架构设计。

## 八、关键原则

### ✅ 要讲

- 用户体验和感受
- 真实的使用场景
- 用户为什么离不开
- 商业模式和增长逻辑

### ❌ 不讲

- 具体技术架构
- 实现细节和算法
- 代码层面的设计
- 专利前的创新点

### 核心策略

```
把技术转化为体验语言：

不说："我们用向量空间存储记忆"
而说："它像人脑一样，把相关的事物连接起来"

不说："链接强度驱动神经元激活"
而说："它会根据和你的亲密度，决定怎么说最合适"

不说："五大驱动系统"
而说："它有自己的'想法'——想知道、想连接、想表达"
```
